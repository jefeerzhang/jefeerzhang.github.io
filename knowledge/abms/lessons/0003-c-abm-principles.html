Lesson 03 · 方法论基础

c-ABM 七条建模原则与 ACE

前两课建立了直觉。现在来掌握 ABM 方法论的"宪法"——Tesfatsion 提出的 c-ABM（Completely Agent-Based Modeling，完全基于主体的建模）的七条建模原则。

什么是 c-ABM？

很多模型虽然包含"Agent"，但其实偷偷引入了外部协调机制（比如一个全局均衡条件、一个中央规划者）。c-ABM 要求：模型中的一切动态结果都必须完全由 Agent 之间的局部交互产生，不借助任何外部"上帝之手"。

核心类比：培养皿实验

把 c-ABM 模型想象成一个培养皿。研究者设定初始条件（投放什么菌种、什么营养），然后观察培养皿里发生了什么。研究者不能伸手进去干预——一切结果都是菌种之间相互作用产生的。

七条建模原则 (MP1–MP7)

以下每条原则用一句话概括核心要求，然后展开解释：

MP1: 主体定义 (Agent Definition)

模型必须明确定义什么是 Agent。Agent 可以是任何实体——个人、公司、政府、甚至天气现象。关键是：每个 Agent 都有一个"局部状态"（local state），描述它在任何时刻的状况。

例子：在一个市场模型中，Agent 可能是买家和卖家，每个 Agent 的局部状态包括它持有的商品数量、资金、当前的报价策略。

MP2: 主体范围 (Agent Scope)

模型中所有有意义的动态过程都必须通过 Agent 的行为和交互来描述。不能偷偷引入一个"社会规划者"或"全局均衡求解器"。

反例：传统经济学模型经常假设存在一个 Walrasian Auctioneer（瓦尔拉斯拍卖人）来调整价格使供需平衡——这违反了 MP2。

MP3: 主体局部建构性 (Agent Local Constructivity)

每个 Agent 在时刻 t 的行为只能基于它自己的局部信息——它不知道全局状态，除非通过与其他 Agent 的交互间接获取。

直觉：你在超市购物时，只能看到你面前的货架和你手里的钱，你不知道全国的库存情况。

MP4: 主体自主性 (Agent Autonomy)

每个 Agent 独立做出决策。没有外部力量直接决定某个 Agent 的行为。

注意：Agent 可以受其他 Agent 影响（通过交流、观察），但最终的决策权在自己手里。

MP5: 系统建构性 (System Constructivity)

系统在任何时刻 t+1 的状态，完全由时刻 t 的所有 Agent 局部状态和它们的交互规则决定。不存在"跳跃到均衡"的捷径。

含义：模型必须是"一步步走"的——每一轮模拟都代表一次真实的交互过程。

MP6: 系统历史性 (System Historicity)

系统的当前状态依赖于它的历史路径。初始条件的不同可能导致完全不同的最终结果（路径依赖）。

例子：一个国家的经济制度深受其历史事件（战争、革命、技术引进）的影响，不能只从"当前最优"来解释。

MP7: 建模者角色 (Modeler as Petri Dish Experimenter)

建模者的角色是设定初始条件和观察，而不是在模拟过程中干预 Agent 的行为。

ACE = c-ABM 在经济系统的应用

Agent-Based Computational Economics (ACE) 就是 c-ABM 方法在经济学领域的特化。它把真实经济视为：

ACE 的核心观点

经济系统是一个开放式的、本地建构的序贯博弈 (Locally-Constructive Sequential Game)。

  • 开放式：没有预设的终点或均衡
  • 本地建构：每个决策基于局部信息
  • 序贯博弈：Agent 轮流行动，当前行动影响未来的可能行动

对照表：传统方法 vs ACE 方法

维度传统均衡分析ACE (c-ABM)
Agent 理性完全理性 / 理性预期有限理性、可学习

本课小结

  • c-ABM 有 7 条建模原则，核心是"一切结果必须从 Agent 局部交互中涌现"
  • ACE 是 c-ABM 在经济学中的应用，视经济为"开放式本地建构序贯博弈"
  • 与传统均衡分析的关键区别：有限理性、无全局协调、路径依赖、异质性
  • 建模者的角色是培养皿实验员：设定条件、观察结果、不干预过程
