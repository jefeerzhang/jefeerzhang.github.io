Lesson 08 · 方法论进阶

模型验证与评价：你怎么知道模型是对的？

你已经学会了设计、实现和运行 ABM 模型。但一个根本性的问题还没解决：你凭什么相信模型的结果是可信的？

ABM 模型面临一个独特的挑战：由于涌现现象的存在，你不能简单地把模型的预测与现实数据做回归比较——模型可能产生了正确的宏观结果，但微观机制完全错了。这就是著名的"同果异因"问题。

四种验证方法

Tesfatsion 在她的方法论文档中区分了四类验证，从最基础到最严格：

1. 输入验证 (Input Validation)

输入验证问的是：模型的输入合理吗？

检查 Agent 的属性、参数、初始条件是否有经验依据。

  • Agent 数量是否与现实匹配？
  • 参数值是否基于经验估计？
  • 初始条件是否反映了研究对象的真实状态？

例子：如果你在模拟一个 100 人的村庄，但你放了 10000 个 Agent——这就违反了输入验证。

这是最基础的验证，但仅此不够——输入合理不代表输出正确。

2. 过程验证 (Process Validation)

过程验证问的是：模型的内部机制合理吗？

检查 Agent 的行为规则、交互方式、学习机制是否符合理论或经验。

  • Agent 的决策规则是否有行为科学依据？
  • 交互结构（网络/空间）是否反映了现实？
  • 时间步进机制是否合适？

例子：如果你的消费者 Agent 在每步都以 100% 的概率选择效用最大的商品——这违反了有限理性假设。过程验证要求你说明为什么 Agent 这样行动是合理的。

3. 描述性验证 (Descriptive Validation)

描述性验证问的是：模型能重现已知的经验规律吗？

让模型运行，把输出的宏观统计特征与经验数据进行比较。

  • 模型产生的价格序列是否与现实价格序列有类似的统计特征（波动率、自相关等）？
  • 模型产生的网络结构是否与现实网络有类似的拓扑特征（度分布、聚集系数等）？
  • 模型产生的不平等指标（基尼系数等）是否与现实数据一致？

注意：描述性验证只能证明模型"看起来对"，不能证明模型"确实对"——因为同果异因问题。

4. 预测性验证 (Predictive Validation)

预测性验证问的是：模型能预测尚未观察到的现象吗？

这是最严格的验证：模型不仅要解释已知的数据，还要事先预测尚未发生的事件或样本外表现。

例子：你用一个 ABM 模型分析电力市场，如果模型准确预测了一种新的拍卖规则实施后会出现的价格波动模式——而这是传统模型无法预测的——这就是强有力的预测性验证。

现实挑战：预测性验证在社会科学中很困难，因为我们很少有机会做真正的"受控实验"来测试预测。

ABM 模型的展示规范

Tesfatsion 的 AModGuide 建议 ABM 论文应包含以下要素：

  1. 模型目的：你试图回答什么问题？
  2. Agent 定义：有哪些类型的 Agent？各自的属性和行为规则是什么？
  3. 环境描述：Agent 在什么空间/网络中交互？
  4. 调度机制：Agent 的行动顺序是怎样的？
  5. 参数选择：参数值从何而来？为什么选这些值？
  6. 实验设计：你做了哪些对照实验？如何控制变量？
  7. 结果展示：统计图表 + 典型运行轨迹
  8. 验证说明：你做了哪种验证？结果如何？

常见的评价陷阱

陷阱问题如何避免
过度拟合调参数让模型"完美"匹配一个特定数据集留出样本外数据做预测验证
同果异因多个不同机制产生相同的宏观结果用过程验证检查机制合理性
单次运行只展示一次运行的结果多次运行，报告统计分布
参数敏感性结果只在特定参数下成立做参数扫描/敏感性分析
黑箱模型模型太复杂无法理解因果关系从简到繁，逐步增加复杂性

测验

Q1: "检查 Agent 的行为规则是否有行为科学依据"属于哪种验证？

输入验证 过程验证 描述性验证 预测性验证

Q2: "同果异因"问题主要威胁的是哪种验证的可信度？

输入验证 过程验证 描述性验证 预测性验证

本课小结

  • ABM 验证分四类：输入验证 → 过程验证 → 描述性验证 → 预测性验证
  • 同果异因是 ABM 最大的方法论挑战——输出正确不等于机制正确
  • 好的 ABM 论文应完整展示模型的目的、Agent、环境、参数、实验和验证
  • 多次运行 + 敏感性分析 + 样本外预测是避免陷阱的关键手段
