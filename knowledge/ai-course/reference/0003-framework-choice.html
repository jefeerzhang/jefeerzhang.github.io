PyTorch vs TensorFlow 选择指南

本课程同时提供两种框架的 Notebook，如何选择？从学习曲线、代码风格和目标场景来判断。

一句话结论

初学者建议：先学 PyTorch。 它的 API 更接近 Python 直觉，调试方便，课程中的大量 notebook 和学术资料也以 PyTorch 为主。学完 PyTorch 后再看 Keras/TensorFlow 会非常快。

对比表

维度PyTorchTensorFlow/Keras
学习曲线 较平缓。动态图机制，和 Python 原生代码几乎一样思考。 Keras 很平缓；底层 TensorFlow 较陡。静态图概念对初学者需要适应。
代码风格 显式定义 forward 和训练循环，控制感强。 Keras 高度封装，几行代码就能训练模型。
调试 可用 Python 标准调试工具（pdb、print），错误栈直观。 Keras 简单模型调试容易；复杂自定义逻辑需要理解 TensorFlow 内部。
学术界/研究 占主导。论文复现和最新模型代码多为 PyTorch。 同样可用，但相对较少。
工业部署 生产部署工具成熟（TorchScript、ONNX、TorchServe），生态增长快。 historically 部署生态更强（TensorFlow Serving、TF Lite），适合移动端和嵌入式。
国内生态 教程丰富，ModelScope、Hugging Face 以 PyTorch 为主。 百度 PaddlePaddle 与 Keras 风格接近，对中文社区友好。

本课程中的对应 Notebook

每节课通常在 PyTorch 和 TensorFlow/Keras 各有一份实现。初学者建议：

  • 挑一种框架完成即可，不要两种同时并行。
  • 遇到某节课只有单一实现时，再切换到另一种框架查漏补缺。
  • 重点理解“数据 → 模型 → 损失 → 优化 → 评估”的通用流程，框架只是实现工具。

建议路径

路径 A（推荐）：PyTorch 主线

1. 第 3 课：运行 PyTorch 版感知器 notebook 2. 第 5 课：用 PyTorch 完成框架简介实验 3. 第 7 课：CNN 选择 PyTorch 版本 4. 第 14 课及以后：NLP 任务使用 PyTorch 实现

路径 B：更喜欢快速上手

1. 第 5 课：从 Keras 顺序模型开始 2. 第 7 课：CNN 用 TensorFlow 版本 3. 后期若需要复现论文或本地跑大模型，再切到 PyTorch
