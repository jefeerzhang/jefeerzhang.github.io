第 10 课：Transformer 与大语言模型

预计 45 分钟 · 目标：理解自注意力机制，了解 BERT/GPT 与提示工程

本课目标： 理解 Transformer 的自注意力机制；了解 BERT（编码器）和 GPT（解码器）的区别；学会写有效的 prompt。

1. 从 RNN 到 Transformer

RNN 按顺序处理序列，有两个问题：无法并行（必须等上一步算完）和长距离遗忘。Transformer 用自注意力机制一次性看整个句子，解决这两个问题。

2. 自注意力机制

对于句子中的每个词，自注意力计算它与其他所有词的关联权重，然后用这些权重对词向量加权求和。

# 自注意力直观理解 句子: "猫坐在垫子上，因为它很累" # "它" 需要知道自己指代 "猫" 而不是 "垫子" # 自注意力通过计算 "它" 与每个词的关联度来实现 # Q(Query): "它在找什么？" K(Key): "我有什么？" V(Value): "我的内容是什么" Attention(Q, K, V) = softmax(Q · Kᵀ / √d) · V

三个关键矩阵：Q（查询）、K（键）、V（值）。每个词同时扮演查询者和被查询者。

3. 多头注意力

用多组 Q/K/V 并行计算注意力，让模型从不同角度关注不同关系（如语法关系、语义关系、指代关系）。

4. 位置编码

自注意力本身没有位置信息（打乱词序结果一样）。位置编码给每个词加上位置信号，让模型知道"谁在前谁在后"。

5. BERT vs GPT

BERTGPT
架构Transformer 编码器Transformer 解码器
方向双向（看全文）单向（只看左侧）
预训练任务完形填空（Masked LM）预测下一个词
擅长理解类任务（分类、NER）生成类任务（写作、对话）
使用方式微调提示工程 + 微调
💡 大语言模型（LLM）： GPT-3/4、BERT 等模型在海量文本上预训练，参数量从亿级到万亿级。它们通过预测下一个词学会了语言、知识和推理能力。

6. 提示工程

通过设计输入提示来引导 LLM 生成更好的输出。好的 prompt 通常包含：角色 + 任务 + 输出格式 + 约束。

# 一个有效的 prompt 结构 """ 你是一位耐心的 AI 入门老师。 请用通俗易懂的中文解释什么是梯度下降， 并举一个生活中的例子。限制在 200 字以内。 """ # 好的 prompt = 角色 + 任务 + 格式 + 约束
