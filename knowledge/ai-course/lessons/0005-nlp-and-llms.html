语言

自然语言处理与大语言模型

理解词嵌入、RNN、Transformer 和提示工程。

本课目标： 理解文本表示方法（从 one-hot 到词嵌入），理解 Transformer 为什么能让大语言模型强大，学会写一个简单的 prompt。

1. 文本表示的演进

神经网络不能直接处理文字，需要把文本变成数字向量。主要方法经历了三代演进：

  • One-Hot（独热编码）： 每个词是一个很长的向量，只有对应位置是 1。缺点：维度高，无法表达词与词的关系。
  • 词袋模型（BOW）/ TF-IDF： 用词的频率表示文档。优点：简单有效。缺点：丢失词序和语义。
  • 词嵌入（Word Embedding）： 把每个词映射为低维稠密向量，语义相近的词在向量空间中距离近。
# 词袋模型示例（scikit-learn）
from sklearn.feature_extraction.text import CountVectorizer

corpus = ["我喜欢猫", "我喜欢狗", "猫和狗都可爱"]
vectorizer = CountVectorizer()
X = vectorizer.fit_transform(corpus)

print(vectorizer.get_feature_names_out())
print(X.toarray())

2. 词嵌入：让语义有距离

Word2Vec 的核心思想是“一个词的意思由它周围的词决定”。训练时会学习：

  • “国王 - 男人 + 女人 ≈ 女王”这类向量运算
  • 语义相近的词向量距离更近

3. 从 RNN 到 Transformer

RNN（循环神经网络）： 按顺序处理单词，适合短句，但长句会出现“遗忘”问题。

Transformer： 使用“自注意力机制”（Self-Attention），一次性看整个句子，计算每个词与其他词的关系。它并行度高、长距离依赖能力强，是现代 NLP 的基础。

# 自注意力直观理解：
# 句子："猫坐在垫子上，因为它很累"
# 模型需要知道 "它" 指代 "猫" 而不是 "垫子"
# 自注意力通过计算词与词之间的关联权重来实现

4. 大语言模型（LLM）与提示工程

大语言模型是在海量文本上训练的 Transformer。它不会“理解”语言，而是通过统计规律生成合理的下一个词。我们可以通过提示（prompt）引导它完成翻译、摘要、问答、写作等任务。

# 一个简单但有效的提示结构
"""
你是一位耐心的 AI 入门老师。请用通俗易懂的中文解释什么是梯度下降，
并举一个生活中的例子。限制在 200 字以内。
"""

# 好的提示通常包含：角色 + 任务 + 输出格式 + 约束
