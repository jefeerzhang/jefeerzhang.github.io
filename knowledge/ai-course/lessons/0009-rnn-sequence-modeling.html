第 9 课：RNN 与序列建模

预计 45 分钟 · 目标：理解 RNN 原理与 LSTM 门控机制，能用 PyTorch 构建序列模型

本课目标： 理解 RNN 如何处理序列数据；理解梯度消失问题和 LSTM/GRU 的门控机制；掌握双向 RNN 和多层 RNN 的概念。

1. 为什么需要 RNN？

前馈网络（MLP、CNN）处理的是固定大小的输入。但很多数据是序列：文本（词序列）、时间序列（股价）、语音（音频帧）。RNN 的核心思想是：用一个"隐藏状态"记住之前看过的内容，逐步处理序列。

2. RNN 原理

在每个时间步 t，RNN 接收当前输入 xₜ 和上一步的隐藏状态 hₜ₋₁，更新隐藏状态并输出：

hₜ = tanh(Wₓ · xₜ + Wₕ · hₜ₋₁ + b) yₜ = Wᵧ · hₜ + b # 关键：Wₓ、Wₕ、Wᵧ 在所有时间步共享同一组参数 # hₜ 像"记忆"，编码了从序列开头到当前步的信息
💡 直觉： 想象你在读一句话，每读一个词就在脑中更新对这句话的理解。RNN 的隐藏状态就像你脑中的"理解"，每读一个词就更新一次。

3. RNN 的梯度消失问题

RNN 在反向传播时要沿时间步展开。当序列很长时，梯度在反复乘以同一组权重后会指数级衰减（消失）或爆炸，导致网络无法学习长距离依赖。

表现： RNN 能记住最近几个词，但"忘了"句子开头的词。

4. LSTM：门控机制

长短期记忆网络（LSTM）通过三个"门"精确控制信息的保留和遗忘：

门作用类比
遗忘门决定从记忆中丢弃什么"这句话的主题变了，忘掉前面的"
输入门决定将什么新信息写入记忆"这个词很重要，记住它"
输出门决定基于记忆输出什么"根据当前理解，做出预测"

GRU（门控循环单元）是 LSTM 的简化版，合并了部分门，参数更少、训练更快，效果通常接近。

5. 用 PyTorch 构建 RNN

rnn_pytorch.pyPython
import torch import torch.nn as nn # 一个简单的 LSTM 文本分类器 class TextClassifier(nn.Module): def __init__(self, vocab_size, embed_dim=100, hidden_dim=128, num_classes=2): super().__init__() self.embedding = nn.Embedding(vocab_size, embed_dim) # batch_first=True: 输入形状为 (batch, seq_len, embed_dim) self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True) self.fc = nn.Linear(hidden_dim, num_classes) def forward(self, x): # x: (batch, seq_len) 整数索引 embedded = self.embedding(x) # (batch, seq_len, embed_dim) # LSTM 返回: output, (h_n, c_n) output, (h_n, c_n) = self.lstm(embedded) # 用最后一步的隐藏状态做分类 return self.fc(h_n[-1]) # (batch, num_classes)

6. 双向 RNN 与多层 RNN

  • 双向 RNN（BiRNN）： 同时从左到右和从右到左处理序列，合并两个方向的隐藏状态。适合需要完整上下文的任务（如 NER）。
  • 多层 RNN（Stacked RNN）： 把多个 RNN 层叠起来，下层输出作为上层输入。层数越多，模型表达能力越强，但也更难训练。
# PyTorch 中设置双向和多层 lstm = nn.LSTM( input_size=100, hidden_size=128, num_layers=2, # 2 层堆叠 bidirectional=True, # 双向 batch_first=True ) # 输出维度: hidden_size * 2（双向拼接）
