# 中文文本预处理基本流程 原文 → 分词 → 去停用词 → 向量化 # 英文："I love cats" → ["I", "love", "cats"] # 中文："我喜欢猫" → ["我", "喜欢", "猫"]（需要分词工具如 jieba）

3. 文本表示方法演进

3.1 One-Hot（独热编码）

每个词是一个很长的向量，只有对应位置是 1。缺点：维度高、无法表达词与词的关系。

3.2 词袋模型（BOW）

用词的频率表示文档，忽略词序。只关心"出现了多少次"，不关心"出现在哪里"。

bow_example.pyPython
from sklearn.feature_extraction.text import CountVectorizer corpus = ["我喜欢猫", "我喜欢狗", "猫和狗都可爱"] vectorizer = CountVectorizer() X = vectorizer.fit_transform(corpus) print(vectorizer.get_feature_names_out()) # 词汇表 print(X.toarray()) # 每行是一个文档的词频向量

3.3 TF-IDF

TF-IDF 在词频基础上加权：一个词在当前文档出现多，但在所有文档中很少见，则权重高。能突出有区分力的词。

from sklearn.feature_extraction.text import TfidfVectorizer tfidf = TfidfVectorizer() X = tfidf.fit_transform(corpus) # TF-IDF 值越高，说明该词对当前文档越重要

4. 词嵌入：让语义有距离

词嵌入把每个词映射为低维稠密向量（如 100-300 维），语义相近的词在向量空间中距离近。

  • Word2Vec： 核心思想"一个词的意思由它周围的词决定"。
  • GloVe： 基于全局词共现矩阵的嵌入方法。

经典例子：国王 - 男人 + 女人 ≈ 女王，说明词向量编码了语义关系。

💡 词袋 vs 词嵌入： 词袋是"稀疏的高维向量"（维度=词汇表大小，大部分为0）；词嵌入是"稠密的低维向量"（维度通常100-300，每个维度都有值）。词嵌入能捕捉语义相似性，词袋不能。
