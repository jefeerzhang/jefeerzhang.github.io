第 11 课：生成模型与强化学习

本课目标： 理解自编码器和 VAE 的编码-解码结构；了解 GAN 的对抗训练思想；掌握强化学习的基本概念和 Q-Learning 原理。

主题 A：生成模型

1. 自编码器（Autoencoder）

自编码器学习把输入压缩成一个低维"潜在表示"，再从中重建出原始输入。两个核心部分：

  • 编码器（Encoder）： 输入 → 潜在空间（压缩）
  • 解码器（Decoder）： 潜在空间 → 重建（还原）

用途：降维、去噪、特征提取。但普通自编码器不能生成新样本——因为潜在空间不连续。

autoencoder.pyPython
import torch.nn as nn class Autoencoder(nn.Module): def __init__(self): super().__init__() # 编码器：784 → 32 维潜在空间 self.encoder = nn.Sequential( nn.Linear(784, 128), nn.ReLU(), nn.Linear(128, 32) ) # 解码器：32 → 784 重建 self.decoder = nn.Sequential( nn.Linear(32, 128), nn.ReLU(), nn.Linear(128, 784), nn.Sigmoid() ) def forward(self, x): z = self.encoder(x) # 压缩 return self.decoder(z) # 重建

2. 变分自编码器（VAE）

VAE 让潜在空间连续且可采样，因此具有生成能力。编码器输出的是分布参数（均值和方差），从中采样再解码。

# 重参数化技巧：让采样可微分 def reparameterize(mu, logvar): std = torch.exp(0.5 * logvar) eps = torch.randn_like(std) # 随机噪声 return mu + eps * std # 采样得到 z

3. 生成对抗网络（GAN）

GAN 用对抗训练：两个网络互相博弈。

  • 生成器： 从随机噪声生成假样本，试图骗过判别器。
  • 判别器： 区分真样本和假样本。

训练过程中两者不断变强，最终生成器能产生逼真样本。

⚠️ GAN 训练难点： 容易出现"模式崩溃"（生成器只会造一种样本）或判别器过强导致生成器学不动。调参比一般模型更讲究。

主题 B：强化学习

4. 强化学习基本概念

强化学习研究智能体在环境中通过试错最大化累计奖励。没有标准答案，只有"做得好不好"的奖励信号。

概念含义CartPole 例子
状态 (State)环境当前描述小车位置、速度、杆角度
动作 (Action)智能体的操作左移 / 右移
奖励 (Reward)即时反馈杆没倒 +1
策略 (Policy)决策规则根据状态选动作
💡 生活类比： 训狗——做对了给零食（正奖励），做错了不给。狗逐渐学会在什么情境下该做什么。

5. Q-Learning

Q-Learning 学习一个 Q 值表 Q(s, a)，表示"在状态 s 下采取动作 a 的期望累计奖励"。

# Q-Learning 更新公式 Q(s, a) ← Q(s, a) + α · [r + γ · max Q(s', a') - Q(s, a)] # α: 学习率 γ: 折扣因子（未来奖励的重要程度） # ε-贪心策略：以概率 ε 随机探索，否则选 Q 值最大的动作

6. 深度强化学习

当状态空间太大时（如游戏画面），Q 表存不下。用神经网络近似 Q 函数就是 DQN。

cartpole_example.pyPython
import gymnasium as gym env = gym.make("CartPole-v1") state, info = env.reset() for step in range(100): action = env.action_space.sample() # 随机动作（先看随机能撑多久） next_state, reward, done, trunc, info = env.step(action) state = next_state if done or trunc: break # 杆倒了，结束本轮
