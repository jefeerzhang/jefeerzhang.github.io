2. PyTorch 核心：Tensor + autograd

pytorch_basics.pyPython
import torch # 创建张量，requires_grad=True 让 autograd 追踪它的梯度 x = torch.tensor([[1.0, 2.0], [3.0, 4.0]], requires_grad=True) # 前向计算 y = (x ** 2).sum() # y = x1² + x2² + x3² + x4² # 反向传播：自动计算 dy/dx y.backward() # 梯度已自动填入 x.grad print(x.grad) # tensor([[2., 4.], [6., 8.]]) ← dy/dx = 2x

3. PyTorch 训练模板

pytorch_train_template.pyPython
import torch import torch.nn as nn from torch.utils.data import DataLoader, TensorDataset # 1. 定义模型 model = nn.Sequential( nn.Linear(784, 128), nn.ReLU(), nn.Linear(128, 10) # MNIST 10 类 ) # 2. 定义损失和优化器 criterion = nn.CrossEntropyLoss() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3) # 3. 训练循环 for epoch in range(10): for xb, yb in dataloader: pred = model(xb) # 前向 loss = criterion(pred, yb) # 算损失 optimizer.zero_grad() # 清旧梯度 loss.backward() # 反向传播 optimizer.step() # 更新参数

5. 过拟合 vs 欠拟合

现象表现原因对策
欠拟合训练集和测试集都差模型太简单加层、加神经元
过拟合训练集好、测试集差模型太复杂/数据太少正则化、Dropout、加数据
理想训练集和测试集都好模型复杂度匹配保持现状
💡 偏差-方差权衡： 欠拟合 = 高偏差（模型能力不够）；过拟合 = 高方差（模型对训练数据太敏感）。目标是找到两者之间的甜点。

6. 正则化方法

# 1. Dropout：训练时随机丢弃一部分神经元 nn.Dropout(p=0.5) # 50% 概率置零 # 2. L2 正则化（权重衰减）：在损失中加入 ||w||² 惩罚 optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4) # 3. 早停（Early Stopping）：验证集 loss 不再下降时停止训练 # PyTorch 需手动实现；Keras 有内置回调 early_stop = tf.keras.callbacks.EarlyStopping(patience=5, restore_best_weights=True)
