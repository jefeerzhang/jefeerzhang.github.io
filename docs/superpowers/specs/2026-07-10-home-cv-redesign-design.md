# 主页与个人简历页统一改版设计

## 项目背景

本次改版覆盖以下两个页面：

- 主页：`https://jefeerzhang.github.io/`
- 个人简历：`https://jefeerzhang.github.io/cv/`

硕士课程站点 `https://jefeerzhang.github.io/master-course/` 属于独立仓库，本次不修改其页面和路由。

## 设计判断

这是面向学生、同行和潜在合作方的学术个人主页，不是产品营销站点。改版应提高身份识别、内容扫描效率和视觉完成度，同时保持学术可信度与内容准确性。

采用“编辑式学术主页”方向，属于定向重构而不是完整品牌推倒重做。

设计参数：

- `DESIGN_VARIANCE: 6`：允许非对称布局，但不使用实验性构图。
- `MOTION_INTENSITY: 3`：仅保留进入、悬停、焦点和按压反馈。
- `VISUAL_DENSITY: 4`：信息充足但保持清晰留白。

## 现状审计

### 主页

- 首屏标题是“课程与资料”，没有把“张剑”作为第一身份信号。
- 页面没有站点导航，主页、CV、课程和技能之间缺少稳定路径。
- 三个主入口采用相同的白色卡片，信息层级单一。
- 工具区只有一个半宽卡片，桌面端右侧形成无意义空位。
- 页面缺少 `main` 和 `nav` 语义地标。
- 页面没有 description、Canonical 和 Open Graph 元信息。
- 生产页面依赖 Google Fonts 外链。

### 个人简历页

- 页面内容可信且完整，但与主页缺少统一的导航和版式系统。
- ORCID 按钮使用高饱和荧光绿和大胶囊造型，视觉权重超过姓名与职务。
- 五个科研数字全部使用悬浮卡片，造成重复容器和视觉噪声。
- 18 条论文记录形成很长的连续列表，桌面与移动端扫描成本较高。
- 页面存在“教育背景、工作经历、研究方向、教学、获奖等板块待补充”的未完成内容。
- 页面没有 `main` 和 `nav` 语义地标，也缺少 SEO 和分享元信息。
- ORCID 使用手写 SVG，字体系统还混入未加载的 Inter。

## 改版目标

- 让“张剑”成为主页首屏的核心识别信息。
- 统一主页和 CV 页的导航、字体、颜色、间距、圆角和交互语言。
- 提高课程、简历、技能和工具入口的差异化与可扫描性。
- 降低 CV 页长列表的阅读负担，不改变论文题目、年份、DOI 和外链。
- 移除未完成占位内容。
- 加强语义结构、键盘操作、SEO、分享预览和资源稳定性。
- 保持静态 HTML 架构，不引入框架或复杂 JavaScript。

## 非目标

- 不修改硕士课程站点。
- 不改变现有页面 URL、锚点和主要外链地址。
- 不编造新的科研成果、合作单位、奖项、论文或项目。
- 不重写论文题目中的学术标点和正式出版信息。
- 不添加虚构头像、虚构照片、假数据或装饰性品牌 Logo。
- 不增加深色模式、滚动劫持、视差、跑马灯或复杂动画。

## 共享视觉系统

### 页面主题

采用统一浅色主题，保持学术文档的稳定阅读感。

建议颜色：

- 页面背景：冷调浅灰纸面。
- 主表面：柔和偏白色。
- 主文字：炭黑。
- 次级文字：中性灰。
- 唯一强调色：深森林绿。
- 提示词优化器原有紫色仅保留在真实工具图标内部，不扩散为页面强调色。

### 字体

- 中文标题使用 `Noto Serif CJK SC`、`Songti SC`、`STSong` 组成的本地系统衬线栈。
- 正文使用 `Noto Sans CJK SC`、`Microsoft YaHei`、`PingFang SC` 组成的本地系统无衬线栈。
- 移除未加载的 Inter 声明。
- 页面不再请求 Google Fonts 或其他远程字体资源。

### 形状与阴影

- 内容卡片统一使用 8px 圆角。
- 图标底板和紧凑控件使用 6px 或 8px 圆角。
- 仅在入口卡片需要层级时使用浅色阴影。
- CV 的正文区、论文列表和指标带主要依靠留白与分隔线组织，不使用重复卡片。

### 动效

- 页面加载时只对首屏和主要入口做短暂透明度与位移动画。
- 链接悬停只改变颜色、边框或轻微位移。
- `:active` 提供轻微按压反馈。
- 所有自动动画遵循 `prefers-reduced-motion`。

## 共享站点导航

主页和 CV 页顶部使用相同导航结构：

- 品牌链接：`张剑 / Jian Zhang`
- 首页
- 个人简历
- 硕士课程
- 自定义技能

设计规则：

- 桌面端单行显示，高度不超过 72px。
- 当前页面使用颜色和下划线表达，不使用装饰圆点。
- 移动端允许横向滚动或紧凑换行，不引入汉堡菜单 JavaScript。
- 页面顶部加入“跳至主要内容”的键盘跳转链接。

## 主页设计

### 首屏

首屏核心内容：

- 英文小标题：`Economics / Research / Teaching`
- H1：`张剑`
- 身份说明：四川农业大学经济学院副教授、硕士研究生导师。
- 简短介绍：聚焦数字经济、绿色发展与应用计量研究，并持续整理教学和研究资料。
- 主操作：查看个人简历。
- 次操作：进入硕士课程。
- 辅助学者链接：ORCID。

首屏采用左对齐编辑式排版，不使用分栏假截图、虚构人物照片或装饰性渐变。

### 入口区域

使用非对称 CSS Grid，而不是三张等宽卡片：

- 硕士课程：占据更大面积，作为学生的主要入口。
- 个人简历：突出身份、论文和 ORCID。
- 自定义技能：使用真实技能内容，链接到 `cv/#skills` 和 GitHub 仓库。
- 工具：提示词优化器使用真实本地图标，作为紧凑工具入口。

每个入口必须有独立的视觉角色，不重复相同卡片结构。

### 页脚

页脚保留 GitHub Pages 托管信息，并增加稳定的 GitHub 和 ORCID 链接。页脚不显示版本号、构建号或虚构状态。

## 个人简历页设计

### 顶部身份区

- 使用与主页一致的站点导航。
- 姓名、职务和机构保持最高视觉层级。
- ORCID 改为紧凑文本链接，使用官方绿色作为局部品牌色，但不使用大型荧光胶囊。
- 删除重复的“返回首页”链接，由统一导航承担返回路径。

### 页面结构

桌面端采用两栏布局：

- 左侧为章节导航，包含个人情况、科研业绩、发表论文和自定义技能。
- 右侧为正文内容。

移动端自动变为单栏，章节导航位于内容顶部并允许紧凑换行。

### 科研业绩

- 保留当前业绩段落与所有真实数据。
- 五个科研指标改为单一数据带，通过列、留白和细分隔线组织。
- 不使用五张独立阴影卡片。

### 发表论文

- 保留中文与英文期刊分组。
- 使用稳定年份列和正文列。
- 年份、论文题目、期刊和 DOI 保持清晰层级。
- 链接获得一致的悬停与焦点状态。
- 不增加筛选器、分页器或 JavaScript 搜索。

### 自定义技能

- 保留现有技能介绍和真实 GitHub 仓库链接。
- 使用现有 `skills_banner.png` 作为辅助视觉，但不让图片取代文字信息。
- 仓库链接和主要操作使用统一的按钮与外链样式。

### 移除内容

删除“其他”区域及其“待补充”文字。未来有真实内容时再增加对应章节。

## 技术结构

### 文件职责

- `assets/site.css`：主页和 CV 页的共享设计令牌、导航、排版、按钮、焦点、响应式和减少动态规则。
- `index.html`：主页语义结构与内容。
- `cv/index.html`：CV 页语义结构与内容。
- `assets/prompt-optimizer.png`：继续作为真实工具图标。
- `cv/skills_banner.png`：继续作为技能区辅助视觉。

静态页面只允许极少量页面专用 CSS，优先复用 `assets/site.css`。

### 数据与链接

页面没有运行时数据流和表单提交。

- 内部链接保持现有 URL。
- 外部链接继续使用 `target="_blank"` 和 `rel="noopener"`。
- 图片使用固定宽高或 aspect-ratio，降低布局偏移。
- 图片加载失败时，标题、说明和链接仍完整可用。

## SEO 与分享信息

两个页面分别加入：

- 准确的 `<title>`。
- 中文 meta description。
- Canonical URL。
- Open Graph title、description、type 和 URL。
- favicon。

不更改现有路由，不影响 `master-course/` 独立站点。

## 无障碍设计

- 使用 `nav`、`main`、`header`、`section` 和 `footer` 语义元素。
- 每页只有一个 H1，标题层级连续。
- 增加跳至主要内容链接。
- 所有交互元素有清晰 `:focus-visible` 状态。
- 正文与背景达到 WCAG AA 对比度。
- 图标按语义使用空 alt 或描述性 alt，避免重复朗读。
- 移动端触控目标不小于 44px。

## 性能要求

- 不引入 React、Tailwind、动画库或图标库。
- 不使用滚动监听或定时动画循环。
- 字体与图片本地加载，避免不必要的第三方阻塞。
- 维持极低 JavaScript 成本。
- Lighthouse 重点检查 Performance、Accessibility、Best Practices 和 SEO。

## 验证方案

### 自动化检查

- 扩展 Python HTMLParser 测试，覆盖两个页面的导航、语义地标、SEO、入口链接和资源路径。
- 检查主页 H1 为“张剑”。
- 检查 CV 页不再包含“待补充”。
- 检查两页均引用共享 `assets/site.css`。
- 检查工具图标和技能图片存在且为有效图片。

### 浏览器检查

- 桌面视口：1440 x 1000。
- 移动视口：390 x 844。
- 检查导航、首屏、非对称入口网格、CV 双栏与单栏切换。
- 检查文本不溢出，元素不重叠，页面无横向滚动。
- 检查键盘导航、焦点状态和所有主要链接。
- 检查控制台无错误或警告。

### 发布检查

- 推送 `master` 后等待 GitHub Pages 成功部署。
- 重新检查线上主页与 CV 页。
- 验证 `master-course/` 路径未受影响。
- 验证线上静态资源返回 200。

## 验收标准

- 主页首屏明确展示“张剑”和学术身份。
- 主页与 CV 页使用同一套导航和视觉系统。
- 主页入口不再是三张重复白卡，工具区不再留下半列空位。
- CV 页 ORCID 权重适当，科研数字不再使用五张卡片。
- CV 页论文列表更易扫描，原始学术信息没有被改写。
- CV 页不再显示“待补充”。
- 两页均具备语义地标、SEO 元信息、键盘焦点和响应式布局。
- 桌面端和移动端无溢出或重叠，控制台无错误。
- GitHub Pages 发布成功，硕士课程站点保持正常。
