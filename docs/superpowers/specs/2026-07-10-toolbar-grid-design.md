# 主页工具栏网格修复设计

## 背景

主页 `https://jefeerzhang.github.io/` 的“工具栏 · Tools”区域目前只有一个宽约 300px 的白色文字卡片。卡片没有图标、用途说明或明确的信息层级，在页面大面积留白中看起来像尚未完成的白色色块。

源码检查确认，这不是图片加载失败：`index.html` 中不存在工具图片节点。现有 CSS 仍保留 `.tool-block img` 样式，说明此前的礼盒图片被删除后，工具卡片结构没有同步完成。礼盒属于通用装饰，与“提示词优化器”的功能识别关系较弱，本次不恢复该图片。

## 目标

- 消除工具区“空白卡片”的未完成感。
- 使用提示词优化器站点的真实图标，建立清晰的工具识别。
- 将工具区改为可持续扩展的卡片网格，以便以后添加多个工具。
- 保持主页现有的米白、深绿、灰色和衬线标题视觉体系。
- 保证桌面端、移动端、键盘操作和图片异常场景均可正常使用。

## 非目标

- 不重构主页其他课程、简历和技能卡片。
- 不引入前端框架、构建工具或运行时 JavaScript。
- 不在生产页面放置虚构的“后续工具”占位卡。
- 不恢复此前使用的礼盒图片。

## 选定方案

采用“可扩展卡片网格”方案。

工具区使用 `.tools-grid` 容器。桌面端显示两列，窄屏设备显示一列。当前只渲染真实存在的“提示词优化器”卡片；后续新增工具时，在网格中加入同结构的卡片即可自动排列。

## 工具卡片结构

每个 `.tool-card` 是一个完整可点击的外链，包含以下内容：

1. 本地工具图标：提示词优化器站点 favicon 转换得到的 PNG。
2. 工具名称：`提示词优化器`。
3. 简短说明：`优化系统提示词与任务指令`。
4. 外链标识：用于表明链接会在新标签页打开。

整张卡片均可点击，链接继续指向：

`https://prompt.always200.com/#/basic/system`

## 视觉设计

- 卡片使用主页现有白色卡面、浅灰边框和克制阴影。
- 图标置于浅绿色方形底板中，与现有 `--accent` 和 `--accent-soft` 色彩变量一致。
- 标题沿用主页的中文衬线字体；说明文字使用现有无衬线正文体系。
- 卡片悬停时轻微上浮，边框与阴影适度增强。
- 外链标识在悬停时轻微位移，但不改变布局尺寸。
- 圆角控制在现有卡片尺度内，不新增不一致的装饰风格。

## 响应式行为

- 桌面端：工具网格为两列，每张卡片占一列。
- 移动端：在适当断点切换为单列，卡片占满可用宽度。
- 卡片内部使用稳定的图标尺寸和网格布局，避免文本或图片加载引发布局跳动。
- 中文标题与说明允许自然换行，不截断重要内容。

## 资源处理

- 从 `https://prompt.always200.com/favicon.ico` 获取真实站点图标。
- 将图标转换为 PNG，并保存为 `assets/prompt-optimizer.png`。
- 生产页面只引用仓库内的本地资源，不热链接外部图片。
- 图片提供明确的 `width`、`height` 和 `alt`，减少布局偏移并支持无障碍阅读。
- 即使图片加载失败，卡片标题、说明和链接仍完整可见，不会重新变成空白块。

## 可访问性与交互

- 卡片使用语义化 `<a>` 元素，不使用 JavaScript 模拟点击。
- 外链保留 `target="_blank"` 和 `rel="noopener"`。
- 增加清晰的 `:focus-visible` 焦点样式，键盘用户可识别当前焦点。
- 悬停动画遵循现有 `prefers-reduced-motion` 设置；减少动态效果时不执行位移。
- 图标为辅助信息，卡片名称和说明承担主要语义。

## 实现范围

预计修改或新增以下文件：

- `index.html`：替换旧 `.tool-block` 样式与结构，加入 `.tools-grid` 和 `.tool-card`。
- `assets/prompt-optimizer.png`：本地工具图标。
- `.gitignore`：忽略本次设计阶段生成的 `.superpowers/` 临时可视化文件。

不修改 `cv/` 页面及其他现有内容。

## 验证方案

### 静态检查

- 确认 HTML 结构有效、图片路径存在、外链地址正确。
- 确认仓库中没有生产页面引用 `.superpowers/` 临时文件。
- 确认没有遗留未使用的 `.tool-block img` 规则。

### 浏览器检查

- 桌面视口检查两列网格、图标、文字层级、悬停和焦点状态。
- 移动视口检查单列布局、文本换行和触控区域。
- 检查页面控制台无错误和警告。
- 检查工具图标成功加载，尺寸稳定且不失真。
- 检查工具卡片跳转到正确的提示词优化器地址。

### 发布检查

- 将实现提交并推送至 `master`。
- 等待 GitHub Pages 部署完成。
- 重新访问线上主页，确认白色空块已被完整工具卡片替换。
- 再次检查桌面端和移动端线上效果及资源加载状态。

## 验收标准

- 工具栏不再显示只有文字的空白白色块。
- 提示词优化器卡片包含真实图标、名称、用途说明和外链标识。
- 桌面端采用两列可扩展网格，移动端采用单列布局。
- 页面视觉与现有主页风格一致，没有恢复无关礼盒装饰。
- 图片失败时卡片仍可理解、可点击。
- 本地和线上页面均无控制台错误，链接和图片均正常。
